### 详尽版（300 字，现场展示用）

> **clausr.ai** 是为认证、检测、检验（TIC）行业从零设计的 AI 合规评估智能体。我们的核心理念是：每个专家的专业经验，都可以被数字化，成为全机构可以积累、可以分享的资产。知识资产化之后，在组织内自由流动，在需要它的任何地方被复用。
>
> 一个干了三十年的审核员，脑子里装着法规体系的理解、边缘案例的判断逻辑、无数次纠正中积累的洞察。这些都是组织最宝贵的资产——如果被数字化，它们就可以同时赋能所有人，而不再受限于一个人的时间和精力。clausr.ai 做的事情，是帮助审核员将这份隐性经验**显式编码为可执行的数字资产**，存储在称为"Skill"的结构化知识文件中。每一个 Skill 承载一个合规领域的完整审核方法论——从步骤定义、法规引用规则，到判定逻辑和持续进化的经验修正。知识的流向也不再只是"老师傅带新人"，而是从有知识的地方，流向需要知识的地方——可以是从高级审核员到初级审核员，可以是从一家机构到整个行业，也可以是从检测环节回溯到设计环节。
>
> 对检测机构的决策者而言，这意味着：报告质量不再取决于当天谁在岗，专家经验变成了公司的资产而非个人的成本。对甲方客户而言，这意味着：每一份检测报告都可以追溯到具体的法规条款，合规结论有据可查、审计就绪。
>
> 技术上，clausr.ai 不是简单地让 LLM 回答问题。我们自研了**分步管道执行架构**：步骤从 Skill 文件动态解析，按类型分派到 LLM 推理、确定性脚本或 LLM+工具三种执行器；引用编译和结论判定在代码层完成，不经 LLM 生成；数值计算通过脚本执行而非 LLM 猜算；知识修正直接写入 Skill 文件，一次修改全局永久生效。
>
> 我们的愿景：**专业知识被数字化之后，可以持续积累、自由流动、不断增值。** 在"超级个体"时代，专业知识的规模化不再靠堆人，靠的是把经验变成可流动的组织资产。

## 四、产品 / 服务详情

### 功能描述

clausr.ai 提供覆盖完整合规评估工作流的一站式能力：

| 功能模块 | 说明 |
|----------|------|
| **多模态证据采集** | 支持 PDF、图片、扫描件、聊天文本等多格式输入，自动归一化为结构化数据 |
| **智能合规评估** | 依据内嵌的 Skill 知识库，自动执行法规适用性判断、合规检查、数值比对 |
| **条款级溯源引用** | 每条结论精确追溯到具体法规条款和原文，引用在代码层编译，无 LLM 幻觉风险 |
| **合规报告自动生成** | 评估完成后自动输出标准化报告，支持 .docx 模板，格式统一、内容可审计 |
| **知识进化系统** | 审核员修正判定后，修改直接写入 Skill 文件，同一 Skill 下所有后续评估自动应用新规则 |

### 技术亮点

- **分步管道执行架构**：步骤从 Skill 文件动态解析，不硬编码；按步骤类型（推理/脚本/工具）分派到专用执行器，强制执行顺序，不依赖 LLM 自觉。
  > 这意味着：**不懂编程的领域专家，在 Skill 文件里写下自己的审核步骤，AI 就严格按这些步骤执行——不跳步、不偷懒、不自作主张。** 专家的方法论直接变成 AI 的工作流程。
- **内置处理器 —— 关键操作不经 AI 之手**：引用文档加载、引用编译、结论判定，通过关键词匹配步骤标题在代码层直接执行，零 LLM 调用。
  > 这意味着：**每一份报告里的法规引用，都是从法规原文库中精确提取的，不是 AI"凭印象"编出来的。** 审计时拿到的每一条引用，都有原文可对。
- **数值计算走脚本，不靠 AI 估算**：合规检测中的数值比对（角度、温度、尺寸范围等），通过调用确定性的 Python 脚本执行，LLM 只负责判断"适用哪条法规"。
  > 这意味着：**每一个数字判断都是精确计算的结果，不会有"差不多""大概"这样的隐患。** 0.57° 和 0.58° 之间就是 PASS 和 FAIL 的差别——AI 不能在这个上面"猜"。
- **知识进化是改文件，不是训模型**：审核员修正判定后，直接修改 Skill 文件的规则章节，系统通过 fuzzy dedup 自动去重，所有后续评估立即应用新规则。
  > 这意味着：**审核员自己就能持续优化系统，不需要找程序员，不需要重新训练 AI。** 今天调整一条判定规则，明天所有评估就按新标准执行。知识进化可追溯、可对比、可回滚。

### 解决的痛点

| 痛点 | clausr.ai 的方案 |
|------|------------------|
| 专业经验缺乏数字化的积累和分享机制 | 经验编码为 Skill 数字资产，在组织内自由流动、持续积累 |
| 不同审核员出具报告标准不统一 | 同一 Skill 驱动的评估流程，输出天然一致 |
| AI 生成的法规引用可能是编造的 | 引用在代码层从法规库精确匹配编译 |
| 审核结果难以应对监管审计 | 每条结论带条款原文，审计线索内建 |
| AI 犯了错不知道怎么改、改了不知道是否生效 | 直接修改 Skill 规则章节，全局生效，可验证 |
| 专业知识只在单点生效，无法赋能全流程 | 知识从有经验的人流向需要知识的环节——从审核到设计、从机构到行业 |

## 五、项目 BP

### 1. 愿景

让专业知识成为可以积累、可以分享、可以增值的数字资产。专家的方法论通过 AI 被放大——赋能团队里的每个人，也流向产业链上需要它的每个环节。

### 2. 问题

认证检测行业不缺人，缺的是让知识流动和积累的载体：
- **经验难以规模化分享**：最有价值的法规理解、判断逻辑、边缘案例经验，存在于审核员的个人实践中——每培养一个新人，都要从头来一遍
- **一致性难保证**：不同审核员对同一标准的理解和执行存在偏差，报告质量取决于当天谁在岗
- **经验的价值被限定在单点**：检测端发现的规律，难以回流到设计端；一家机构积累的教训，难以转化为行业的公共知识

### 3. 解决方案

clausr.ai —— 帮助审核员将专业方法论编码为结构化 Skill 文件，由 AI 按步骤严格执行合规评估。知识被数字化后，每一条结论天然可追溯至法规原文，每一次经验修正永久全局生效，每一份知识都能被分享、被复用、被积累。

### 4. 差异化

| 维度 | 通用 AI Chatbot | clausr.ai |
|------|----------------|-----------|
| 知识载体 | 提示词工程 | 结构化 Skill 文件 |
| 执行方式 | LLM 自行排序 | 分步管道强制执行 |
| 引用来源 | LLM 生成（可能编造） | 代码层从法规库精确编译 |
| 数值计算 | LLM 语言推理 | Python 确定性脚本 |
| 知识进化 | 黑盒微调 | 显式修改 Skill 文件，可审计可回滚 |
| 行业适配 | 通用，无领域深度 | 为 TIC 行业从零设计 |
